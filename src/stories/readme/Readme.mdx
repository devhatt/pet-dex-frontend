# Read-me

<br />

## O que é o Storybook?

O Storybook permite que os desenvolvedores criem e visualizem componentes de forma isolada,
facilitando o processo de desenvolvimento e teste de cada elemento da interface. Com o Storybook,
os desenvolvedores podem criar uma biblioteca de componentes reutilizáveis, documentar suas
funcionalidades e interações, e testar sua aparência e comportamento em diferentes estados e
contextos. Isso ajuda as equipes de desenvolvimento a garantir a consistência visual e funcional
em todo o aplicativo, ao mesmo tempo em que acelera o processo de desenvolvimento e colaboração
entre os membros da equipe.

<br />
<br />

## Para que iremos usar o Storybook no contexto do Petdex?

Inicialmente, planejamos utilizar o Storybook principalmente como uma plataforma para visualização
e documentação dos componentes que compõem nossa interface de usuário.

<br />

## Criando um story

Os componentes da interface do usuário são definidos em arquivos separados dentro da pasta <code>"src/stories"</code>,
cada um representando um elemento específico, como botões, caixas de texto, ou qualquer outra parte da UI.

O arquivo do componente deve obrigatoriamente ter o sufixo <code>.stories.js</code>,
exemplos: <code>src/stories/Button.stories.js</code>, <code>src/stories/Carousel.stories.js</code>.
